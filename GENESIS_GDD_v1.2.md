## 一、游戏概述

### 核心理念

《万物起源》是一款基于同心圆旋转棋盘的合成类休闲小游戏。玩家在一个不断旋转的圆形棋盘上，通过"选中 → 点击相邻同级"的两步交互合成元素，将它们从最微小的 Quark 一步步升级成 Galaxy，体验从微观到宏观的宇宙演化之旅。

### 第一版（MVP）范围（v1.3 更新）

**包含：**
- 同心圆旋转棋盘（核心 + 三层圈层，37 格）
- 10 级元素合成链路（Quark → Galaxy）
- **两步点击交互**（选中 + 连接线提示 + 再点合成）
- **合成动画**（聚合 → 爆发 → 新元素弹出）
- **合成后分裂** + **定时自动分裂**两套并行机制
- 核心吸附升级的完整游戏循环
- 基础计分系统 + Combo 连锁倍率
- **统一配置系统**（`js/config.js` 可调参数）
- 历史最高分本地持久化
- 无时间限制、无关卡、统一旋转速度

**不包含（后续版本迭代）：**
- 好友排行榜
- 合成图鉴
- 里程碑成就
- 特殊元素 / 道具
- 难度递增系统（旋转速度固定）
- Combo 额外奖励机制（当前只有倍率，无道具解锁）

### 目标用户

- 全年龄段（6-60+ 岁），不需要强反应力
- 休闲玩家：碎片时间快速上手，单局无时间限制
- 科普爱好者：合成链路基于真实科学逻辑

### 技术平台

- **平台：** 微信小程序 / 小游戏
- **开发方式：** Vibe Coding（AI 辅助开发）
- **技术栈：** 原生 JavaScript + Canvas API（不使用游戏引擎）
- **视觉风格：** 低多边形几何风 + 粒子效果，暗色背景，简洁科技感

## 二、棋盘结构

### 同心圆轨道系统

棋盘由一个中心核心 + 三层同心圆圈层组成，总计 **37 个格子**。视觉上类似电子轨道 / 太阳系结构，与"从微观到宏观"主题天然契合。

| 圈层 | 格子数 | 旋转方向 | 说明 |
|------|--------|----------|------|
| 核心 Core | 1 | 固定不动 | 始终位于中心，显示当前核心等级，合成后分裂源头 |
| 内圈 Inner | 6 | 顺时针 ↻ | 合成后分裂优先在此生成 |
| 中圈 Mid | 12 | 逆时针 ↺ | 与内/外圈反向旋转；**定时分裂目标池之一** |
| 外圈 Outer | 18 | 顺时针 ↻ | 最大的合成空间；**定时分裂目标池之一** |

**总格子数：1 + 6 + 12 + 18 = 37 格**

### 旋转机制

- 内圈与外圈顺时针旋转，中圈逆时针旋转
- 三层交错旋转使得元素的相邻关系持续变化
- **第一版使用统一的旋转速度**，后续版本可调整

### 相邻定义

元素只能与"相邻"的同级元素合成。相邻规则如下：

- **同圈相邻：** 同一圈层内左右紧挨着的两个格子
- **跨圈相邻：** 上下圈层中径向对齐的格子（因旋转而动态变化）
- **核心相邻：** 内圈 6 个格子始终与核心相邻（用于吸附判定；连接线不绘制核心）

> ⚠️ **注意：** 跨圈相邻关系因旋转而不断变化，这是策略性的核心来源。相邻判定需要基于当前角度实时计算。

## 三、元素等级体系

合成链路基于真实的从微观到宏观科学逻辑，采用线性合成（A + A = B）。

| 等级 | 名称 | 色调 | 视觉特征 | 科学含义 |
|------|------|------|----------|----------|
| Lv.1 | Quark | 淡紫 | 极小圆点 + 微弱粒子 | 基本粒子，物质最小单元 |
| Lv.2 | Proton | 紫色 | 较大圆形 + 散射粒子 | 质子，由 Quark 组成 |
| Lv.3 | Atom | 蓝色 | 核心 + 轨道环 | 原子，有电子轨道 |
| Lv.4 | Molecule | 蓝色 | 双球连接结构 | 分子，原子的组合 |
| Lv.5 | Cell | 青绿 | 同心圆 + 核心点 | 细胞，生命基本单元 |
| Lv.6 | Organism | 绿色 | 不规则多边形 + 点缀 | 生物体，多细胞组织 |
| Lv.7 | Land | 黄绿 | 大型多边形 + 层次感 | 大地 / 生态系统 |
| Lv.8 | Planet | 蓝色 | 同心圆层 + 地表多边形 | 行星，大气层包裹地表 |
| Lv.9 | Star | 金色 | 发光核心 + 射线光芒 | 恒星，自发光的天体 |
| Lv.10 | Galaxy | 深紫+金 | 旋涡结构 + 散布光点 | 星系，宇宙最大结构 |

### 视觉演进逻辑

- **尺寸：** 低级元素小（半径 8 + level × 1.2 px），高级元素大
- **色彩：** 紫 → 蓝 → 绿 → 金，从冷色微观到暖色宏观
- **粒子效果：** 低级随机漂浮，中级轨道环绕，高级光芒辐射

## 四、核心玩法循环

### 初始状态

游戏开始时，棋盘上只有核心位置的一个 **Lv.1 Quark**。其余 36 个格子全部为空。三层圈层开始旋转。

开局瞬间核心连续分裂 **4 次**（间隔 200ms），在内/中/外圈依优先级放置 4 个 Lv.1 元素。期间输入锁定。

### 游戏循环流程（v1.3 更新）

```
Step 1 — 核心分裂（两套机制并行独立运行）
  A. 合成后分裂：玩家完成一次合成后，核心排队分裂 1 个 Lv.1 元素，
     落点依"优先相邻空位"规则（P1-P5）。
  B. 定时自动分裂：游戏开始后进入 30 秒安全期；之后按核心等级查表
     获取间隔（见第九章），每间隔到点前 1.5 秒核心开始前摇呼吸，
     到点从"中圈 + 外圈所有空位"中真随机选一格分裂 Lv.1。
     合成后分裂和定时分裂互不干扰，玩家可在前摇/飞行期间继续合成。

Step 2 — 轨道旋转
  三层圈层持续自动旋转，元素的相邻关系不断变化。
  玩家观察并等待最佳合成时机。

Step 3 — 两步点击合成
  a) 第一次点击一个有元素的格子 → 进入"选中态"：
     该格子出现发光环 + 脉冲呼吸 + 向 4 个几何邻居（同圈左右
     + 跨圈对齐上下）画连接线。连接线实时跟随旋转，相邻+同级
     元素的连接线高亮（元素副色粗线），其余暗淡（灰色细线）。
  b) 第二次点击：
     · 点同一格 → 取消选中
     · 点相邻+同级的格子 → 触发合成
     · 点其他格子（不相邻/不同级）→ 切换选中到新格子，不抖动不报错

Step 4 — 合成动画
  被选中元素和目标元素同时向两者中间点滑动（ease-out 约 300ms，
  同时缩小到 0.7×），聚合瞬间粒子爆发，新的更高级元素从中点回
  到选中格子位置，先放大到 1.2× 再回到 1.0×（约 100ms）。
  动画总时长约 400ms，期间输入锁定。

Step 5 — Combo 连锁
  合成完毕瞬间，系统自动检查新元素是否与其他同级相邻。若是，
  自动再次触发合成动画（无需玩家再点）。每一步 combo 递增，
  分数按 combo 倍率累加。连锁期间输入持续锁定。

Step 6 — 吸附升级
  连锁终止后，系统检查棋盘上是否存在与核心同等级的元素。若有，
  该元素沿径向线飞向核心并融合，核心升级（尺寸变大、颜色切换）。
  吸附可级联（核心升级后如仍匹配则继续吸附）。

Step 7 — 循环
  回到 Step 1。合成后分裂队列会补上此次分裂，死锁保险保证
  Lv.1 总数（棋盘 + 飞行 + 队列）始终 ≥ 2 可合成。
```

### 分裂位置规则

**合成后分裂（优先级链）：**
```
Step 0  内圈无 Lv.1 + 内圈有空位 → 任意内圈空位（开局种子）
P1      内圈同圈相邻于 Lv.1 的空位
P2      中圈同圈相邻于 Lv.1 的空位
P3      中圈无 Lv.1 但内圈有 Lv.1 → 最对齐内圈 Lv.1 的中圈空位
P4      外圈同圈相邻于 Lv.1 的空位
P5      外圈无 Lv.1 但中圈有 Lv.1 → 最对齐中圈 Lv.1 的外圈空位
都失败  触发游戏结束（棋盘已满）
```

**定时分裂（真随机）：**
```
目标池 = 中圈所有空位 + 外圈所有空位（排除 reserved）
均匀随机抽取一格。不使用"优先相邻空位"规则（有意的挑战性设计）。
如果中/外圈都满则跳过本次分裂，不触发 game over；下一轮继续。
```

### 吸附规则

- **任何圈层**的同级元素都能被核心吸附，不限于内圈
- 吸附时触发动画：元素沿径向线飞向核心，融合后核心变大变亮
- 吸附后该格清空

### 失败条件

```
棋盘上所有 37 个格子全部被占满，且队列/飞行/合成动画全部清空 → 游戏结束
```

游戏没有时间限制。只要棋盘没有填满，玩家可以无限玩下去。

### 终极目标

将核心从 Lv.1 Quark 升级到 Lv.10 Galaxy。

## 五、操作方式与反馈

### 基本操作

玩家的唯一操作是**点击**。v1.3 之后采用"两步点击"交互：

1. **第一次点击** → 选中该格子（进入选中态）
2. **第二次点击** → 根据第二次点击的目标决定行为：
   - 点同一格子 → 取消选中
   - 点相邻+同级格子 → 触发合成动画
   - 点其他格子 → 切换选中到新格子

### 选中态视觉

| 元素 | 效果 |
|------|------|
| 被选中格子 | 外围发光环（元素副色，圆形渐变）+ 白色高光描边 |
| 被选中元素本体 | 脉冲呼吸缩放 1.0 ↔ 1.15，周期约 1 秒 |
| 4 条连接线 | 从选中格子指向 4 个"几何邻居"（同圈左右 + 跨圈对齐上下）<br>**内圈** 最多 3 根（无"朝核心"方向，不连核心）<br>**中圈** 4 根<br>**外圈** 最多 3 根（无"远离核心"方向） |
| 连接线状态 | · 相邻 + 同级：高亮元素副色，线宽 2.5px，α 0.85<br>· 其他：暗灰色（rgba 255,255,255,.25），线宽 1px，α 0.55 |
| 实时更新 | 每帧重算端点位置和高亮状态，跟随圈层旋转 |

### 合成动画

| 阶段 | 时长 | 行为 |
|------|------|------|
| 聚合 | 约 300ms（18 帧） | 两元素从各自位置向中点滑动（ease-out cubic），半径从 1.0× 缩小到 0.7× |
| 爆发 | 瞬间 | 中点粒子爆发（新元素主色 + 副色各一组） |
| 弹出 | 约 100ms（6 帧） | 新元素从中点滑回到第一步选中的格子位置，半径 0.5× → 1.2× → 1.0× |

动画期间输入锁定。Combo 连锁时，每一步都走完整动画流程。

### 其他反馈

| 时机 | 反馈效果 |
|------|----------|
| 合成 | 聚合动画 + 中点粒子爆发 + 新元素弹出 |
| 吸附 | 元素沿径向线飞向核心 + 粒子尾迹 + 核心脉冲 |
| Combo | 显示 combo 计数器（金色 "COMBO x{n}!"） |
| 核心分裂 | 核心脉冲放大 +25% 发光 15 帧 |
| 定时分裂前摇 | 核心外层暖色（#EF9F27）慢呼吸环，1.5s 完成一次 0→1→0 |

## 六、计分系统

### 基础分数

```
每次合成得分 = 合成后元素等级 × 10

示例：
  两个 Lv.1 合成出 Lv.2 → 得 20 分
  两个 Lv.5 合成出 Lv.6 → 得 60 分
```

### Combo 加成

```
连锁时分数按倍数递增：
  第 1 次合成：基础分 × 1
  第 2 次连锁：基础分 × 2
  第 3 次连锁：基础分 × 3
  以此类推……
```

### 吸附升级奖励

```
核心每次升级额外奖励 = 新等级 × 100

示例：
  核心 Lv.1 → Lv.2：奖励 200 分
  核心 Lv.9 → Lv.10：奖励 1000 分
```

### 最终得分

```
最终得分 = 基础合成分累计 + Combo 加成累计 + 吸附升级奖励累计
```

### 历史最高分

- 存储于微信本地存储 `wx.setStorageSync`，键名 `genesis_high_score`
- 游戏结束时自动比较并更新；"再来一局"不清除
- 顶部 UI 左上角金色小字显示，带皇冠图标

## 八、技术备注

### 渲染方案

- 使用微信小游戏 Canvas API 渲染棋盘和元素
- 旋转动画：使用 `requestAnimationFrame` 游戏循环，每帧更新元素在圆弧上的角度位置
- 每帧假设 60fps；所有时长参数在 `js/config.js` 中以秒/毫秒表示，通过 `secondsToFrames` / `msToFrames` 换算

### 关键数据结构（v1.3 更新）

```javascript
// 格子数据结构
slot = {
  ring: 'inner' | 'mid' | 'outer',  // 所在圈层
  slotIndex: Number,                  // 在该圈层中的索引
  level: Number | null,               // 当前元素等级，null 表示空
  reserved: Boolean,                  // 已被分裂队列锁定（飞行中）
  mergeAnimating: Boolean,            // 正在合成动画中（被 drawSlots 跳过）
  baseAngle: Number                   // 初始角度（弧度）
}

// 核心数据结构
core = {
  level: Number  // 当前核心等级（1-10）
}

// 棋盘实例状态（主要）
board = {
  slots: slot[],                   // 37 格
  core: core,
  rotation: { inner, mid, outer }, // 弧度偏移
  rotationSpeeds: { inner, mid, outer },

  // 合成后分裂
  queuedSplits: Number,
  splitTimer: Number,
  flyingElements: Array<{ startX, startY, targetSlot, frame, totalFrames }>,
  corePulse: Number,

  // 定时分裂
  gameFrame: Number,                    // 游戏帧计数（不因暂停重置）
  safePeriodFrames: Number,             // 由 config 计算得出
  warningFrames: Number,
  timedSplitScheduledFrame: Number|null,// 下次触发帧号
  timedSplitWarningProgress: Number,    // 0..1 前摇进度

  // 两步点击 + 合成动画
  selectedSlot: slot|null,
  mergeAnimations: Array<{             // 当前进行中的合成动画
    slotA, slotB,
    sourceLevel, newLevel,
    frame, convergeFrames, popFrames, totalFrames,
    burstFired
  }>,

  // 输入锁
  initialSplitsComplete: Boolean,
  inputLocked: Boolean
}
```

### 相邻判定逻辑

```
同圈相邻：
  同一圈层内，两个格子的 slotIndex 差值为 1（或首尾相接）

跨圈相邻：
  仅 inner↔mid / mid↔outer 允许
  两格子的当前角度差 < 阈值（较大圈层单格角度 × 0.5）

连接线邻居（getNeighborsForConnection，v1.3 新增）：
  同圈左 + 同圈右（固定不随旋转变化）
  跨圈朝核心侧 → 目标圈中"当前角度最接近本格"的那个
  跨圈远离核心侧 → 同上
  内圈无"朝核心侧"，外圈无"远离核心侧"（不连核心）
```

### 粒子效果

```javascript
particle = {
  x, y, vx, vy, radius, color, alpha, life, maxLife
}

// 触发时机：
// - 合成中点：spawn() 主色/副色两组粒子爆发
// - 合成后分裂发射：spawn() 核心位置一小圈
// - 定时分裂发射：spawn() 核心位置更大一圈（主色，与初始分裂做视觉区分）
// - 飞行尾迹：spawnTrail() 沿路径
// - 吸附尾迹：spawnTrail() 沿径向
// - 装饰粒子：每 30 帧为有元素的格子 spawnDecoration() 一次
```

### 性能注意

- 同屏粒子上限 200
- 合成动画每帧新增绘制 ≈ 4 条连接线 + 1 个发光环 + ≤2 个动画中元素 + 核心呼吸叠加，负载远低于粒子上限
- 连接线每帧重算：4 次跨圈角度查询 + 4 次 moveTo/lineTo，Canvas 绘线开销可忽略

### 点击检测

```
触摸坐标与元素位置的距离判定：
  distance = sqrt((touchX - elementX)² + (touchY - elementY)²)
  如果 distance < elementRadius + 触摸容差（10px），则命中

跳过条件：
  · slot.level === null（空格）
  · slot.reserved === true（飞行未落地）
```

## 九、定时分裂机制

### 设计目的

为长局增加压力曲线、让棋盘逐步收紧，同时随核心等级提高而加快。合成后分裂依赖玩家操作节奏，定时分裂则是独立的"外部时钟"，逼迫玩家始终保持合成效率。

### 触发规则

1. **安全期**：游戏开始后的前 30 秒（`safePeriodSeconds`）不触发定时分裂，玩家有时间熟悉棋盘
2. **间隔（按核心等级查表）：**

   | 核心等级 | 间隔 |
   |----------|------|
   | Lv.1 - Lv.3 | 20 秒 |
   | Lv.4 - Lv.5 | 15 秒 |
   | Lv.6 - Lv.7 | 12 秒 |
   | Lv.8 - Lv.10 | 8 秒 |

3. **连续运行**：计时器在游戏进行中持续运行，**不因玩家合成而重置**
4. **前摇 1.5 秒**（`preSplitWarningSeconds`）：每次触发前核心外层出现暖色（#EF9F27）呼吸环，吸气-呼气一次完整周期正好跨越 1.5 秒。目的是"醒目但不打扰"，让玩家用余光察觉即将分裂，但不会干扰当前操作
5. **发射瞬间**：前摇结束，元素从核心飞到目标格（18 帧 ≈ 300ms）

### 目标选择

- 目标池 = 中圈所有空位 + 外圈所有空位（排除 `reserved`，即飞行未落地的）
- **真随机均匀抽取**一格
- 不使用"优先相邻空位"规则 — 这是有意的挑战性设计，逼迫玩家应对"陌生空位突然出现一个 Lv.1"
- 中圈和外圈都满时跳过本次分裂（不触发 game over）；下一轮到点继续尝试

### 分裂元素等级

永远是 **Lv.1**。

### 与合成后分裂的关系

- 两套机制**并行独立运行**，互不替换
- 共用底层 `flyingElements[]` 队列，共用 `onLand` 落地粒子回调
- 定时分裂期间玩家可以继续选中/合成（不锁输入）；只有合成动画本身会锁输入

## 十、配置系统

### 文件位置

`js/config.js` — 集中管理所有可调参数，通过 CommonJS 导出 `GAME_CONFIG` 单例。

### 配置结构

```javascript
const GAME_CONFIG = {
  // 定时自动分裂
  timedSplit: {
    safePeriodSeconds: 30,
    preSplitWarningSeconds: 1.5,
    intervalByLevel: {
      '1-3': 20, '4-5': 15, '6-7': 12, '8-10': 8,
    },
  },
  // 开局连续分裂
  initialSplit: {
    count: 4,
    intervalMs: 200,
  },
  // 交互
  interaction: {
    showConnectionLines: true,
    selectionPulseEnabled: true,
  },
  // 合成动画
  mergeAnimation: {
    convergeFrames: 18,  // 300ms @ 60fps
    popFrames: 6,        // 100ms @ 60fps
  },
};
```

### 辅助函数

- `secondsToFrames(seconds)` — 秒转帧（60fps 基准）
- `msToFrames(ms)` — 毫秒转帧
- `getTimedSplitInterval(coreLevel)` — 按核心等级查表返回秒数；范围键格式 `'lo-hi'`

### 访问方式

```javascript
const { GAME_CONFIG, secondsToFrames, getTimedSplitInterval } = require('./config');
```

每个需要读配置的模块（board.js / renderer.js）在文件顶部 require。config 是叶子节点模块，无其他依赖。

### 作用域说明

本轮（v1.3）仅把三项新功能明确需要的参数放入 `GAME_CONFIG`。旋转速度、元素颜色、脉冲帧数、飞行帧数、棋盘半径比、触摸容差等"尚未必要"的常量仍保留在各自模块，后续如需批量整理再迁移。
