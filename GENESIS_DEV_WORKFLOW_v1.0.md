## 第一阶段：环境搭建

按顺序安装以下工具，每步完成后再进行下一步。

---

### 1.1 安装 Node.js

Node.js 是 JavaScript 的运行环境，很多开发工具依赖它。

**步骤：**
1. 打开浏览器，访问 [nodejs.org](https://nodejs.org)
2. 下载 **LTS（长期支持）** 版本的 macOS 安装包
3. 双击安装，一路点"继续"即可
4. 安装完成后，打开终端（Terminal），输入以下命令验证：

```bash
node --version
npm --version
```

如果显示版本号（如 `v20.x.x`），说明安装成功。

---

### 1.2 安装 Cursor

Cursor 是你的主力代码编辑器，内置 AI 辅助编码功能。

**步骤：**
1. 访问 [cursor.com](https://cursor.com)，下载 macOS 版
2. 安装并打开，登录账号（可用 Google 账号登录）
3. 进入设置（Settings），确认 AI 模型已启用

**Cursor 核心快捷键：**

| 快捷键 | 功能 |
|--------|------|
| `Cmd + K` | 在代码文件中让 AI 生成或修改代码 |
| `Cmd + L` | 打开 AI 对话框，用自然语言描述需求 |
| `Cmd + I` | Composer 模式，处理多文件操作 |

> 💡 **使用技巧：** 把设计文档 `GENESIS_GDD_v1.2.md` 拖入 Cursor，对话时用 `@GENESIS_GDD_v1.2.md` 引用，AI 会理解完整的游戏设计背景再写代码。

---

### 1.3 安装 Claude Code

Claude Code 是 Anthropic 的命令行 AI 工具，适合处理生成项目框架、批量重构等大操作。

**步骤：**
1. 打开终端，输入以下命令安装：

```bash
npm install -g @anthropic-ai/claude-code
```

2. 安装完成后，输入 `claude` 启动，首次使用需要登录 Anthropic 账号
3. 登录成功后，在项目目录中输入 `claude` 并用自然语言描述任务即可

**Claude Code vs Cursor 的分工：**

| 工具 | 适合场景 |
|------|----------|
| Cursor | 日常编码：写新功能、调试、小范围修改 |
| Claude Code | 大操作：一次性生成整个项目框架、批量重构多文件 |

---

### 1.4 安装微信开发者工具

微信开发者工具是调试和预览小游戏的必备工具。

**步骤：**
1. 访问微信开发者工具下载页面：
   `developers.weixin.qq.com/miniprogram/dev/devtools/download.html`
2. 下载**稳定版（Stable）** macOS 版本
3. 安装并打开，用微信扫码登录
4. 登录后会看到项目列表界面，**先不要创建项目**，等下一阶段

---

### 1.5 安装 Figma

Figma 用于设计游戏 UI 和元素样式。

**步骤：**
1. 访问 [figma.com](https://figma.com)，注册账号并下载桌面版
2. 根据设计文档中的元素样式参考，在 Figma 中绘制 10 个等级的元素图标
3. 导出格式：每个元素导出为 PNG，背景透明，建议尺寸 2x（如 128×128px）

## 第二阶段：创建项目

### 2.1 用 Claude Code 生成项目框架

这是整个开发中最关键的一步。用 Claude Code 一次性生成完整的微信小游戏项目结构。

**步骤：**

1. 在终端中创建项目文件夹：

```bash
mkdir ~/genesis-game && cd ~/genesis-game
```

2. 把设计文档放入项目文件夹：

```bash
cp ~/Downloads/GENESIS_GDD_v1.2.md ~/genesis-game/
```

3. 启动 Claude Code：

```bash
claude
```

4. 将以下提示词（结合设计文档内容）发给 Claude Code：

```
请参考 @GENESIS_GDD_v1.2.md，帮我创建一个微信小游戏项目。

技术要求：
- 使用原生 Canvas API，不使用任何游戏引擎
- 语言：JavaScript
- 按以下文件结构组织代码

请生成完整的项目文件结构和基础代码：
- game.js（游戏入口和主循环）
- game.json（小游戏配置）
- project.config.json（项目配置，appid 先留空）
- js/board.js（棋盘数据结构与逻辑）
- js/renderer.js（Canvas 渲染）
- js/particles.js（粒子效果系统）
- js/input.js（触摸事件处理）
- js/score.js（计分系统）
- images/（空文件夹，用于放元素图片）
```

**Claude Code 生成的项目结构：**

```
genesis-game/
├── game.js              ← 游戏主逻辑（棋盘、旋转、合成）
├── game.json            ← 小游戏配置
├── project.config.json  ← 项目配置（填入你的 AppID）
├── GENESIS_GDD_v1.2.md  ← 设计文档
├── js/
│   ├── board.js         ← 棋盘数据结构与逻辑
│   ├── renderer.js      ← Canvas 渲染（画圆、画元素）
│   ├── particles.js     ← 粒子效果系统
│   ├── input.js         ← 触摸事件处理
│   └── score.js         ← 计分系统
└── images/              ← 元素图片（从 Figma 导出后放这里）
```

> ⚠️ **重要：** 在 `project.config.json` 中找到 `appid` 字段，替换为你的微信小游戏 AppID。可以在 [mp.weixin.qq.com](https://mp.weixin.qq.com) 后台找到。

---

### 2.2 在微信开发者工具中导入项目

1. 打开微信开发者工具
2. 点击"导入项目"，选择 `~/genesis-game` 文件夹
3. 项目类型选择"小游戏"
4. 填入你的 AppID
5. 点击"导入"，等待编译完成

如果一切顺利，你应该能在模拟器中看到一个黑色背景上的圆形棋盘。

> 💡 **日常工作流：** 从现在开始，你的工作流是：在 Cursor 中写代码 → 保存 → 微信开发者工具自动刷新预览。两个工具并排放在屏幕上。

## 第三阶段：核心玩法开发

按以下顺序逐步实现，**每完成一步就在模拟器中测试**，确认没问题再进行下一步。

---

### 3.1 画出棋盘

用 Cursor 打开 `js/renderer.js`，让 AI 帮你实现：

- 暗色背景
- 三层同心圆环（内圈、中圈、外圈）
- 圈层上的格子位置标记（可以先用空心圆表示）
- 中心的核心元素

**Cursor 提示词示例：**
```
请帮我在 renderer.js 中实现棋盘绘制。棋盘是三层同心圆，
内圈 6 格、中圈 12 格、外圈 18 格。每个格子是圆弧上的
一个位置。中心有一个固定的核心元素。暗色背景。
参考 @GENESIS_GDD_v1.2.md 中的棋盘结构章节。
```

**验证：** 模拟器中能看到三层同心圆和格子位置标记。

---

### 3.2 实现旋转

让三层圈层开始旋转：

- 内圈 + 外圈：顺时针
- 中圈：逆时针
- 使用 `requestAnimationFrame` 做游戏循环

**Cursor 提示词示例：**
```
请实现圈层旋转。使用 requestAnimationFrame 做游戏循环。
内圈和外圈顺时针旋转，中圈逆时针旋转。每个格子的实际
位置 = 基础角度 + 旋转偏移量。旋转速度先设为常量，后续可调。
```

**验证：** 格子在圆弧上流畅旋转，无卡顿。

---

### 3.3 实现核心分裂

**逻辑：**
```
游戏开始时 → 核心自动分裂第一个 Lv.1 元素到内圈空位
每次合成完成后 → 核心自动分裂一个 Lv.1 元素

分裂位置优先级：内圈空位 → 中圈空位 → 外圈空位
```

**验证：** 游戏开始时内圈出现一个 Lv.1 元素，合成一次后出现下一个。

---

### 3.4 实现点击合成

这一步是核心难点，分两个部分：

**Part A：点击检测**
```
触摸坐标命中判定：
  distance = sqrt((touchX - elementX)² + (touchY - elementY)²)
  if distance < elementRadius + 10px → 命中

注意处理 Canvas 坐标缩放：
  realX = touchX * (canvas.width / canvas.clientWidth)
  realY = touchY * (canvas.height / canvas.clientHeight)
```

**Part B：相邻判定**

> ⚠️ **这是最难的部分。** 跨圈相邻需要实时计算当前角度。

```
同圈相邻：slotIndex 差值为 1（或首尾相接）

跨圈相邻：两个格子当前角度差 < 阈值
  阈值 ≈ 该圈层单格所占角度的 0.5 倍
  需要每帧实时计算，因为圈层在旋转

核心相邻：内圈所有格子始终与核心相邻
```

**Cursor 提示词示例：**
```
请实现点击合成逻辑。参考 @GENESIS_GDD_v1.2.md 中的相邻定义章节。
重点：跨圈相邻需要基于当前角度实时计算，因为圈层在旋转。
同圈左右相邻 + 跨圈径向对齐才算相邻。
合成后元素留在原位，不自动迁移。
```

**验证：** 点击有相邻同级元素的格子，两个元素合并为一个更高级元素。

---

### 3.5 实现 Combo 连锁

```
合成完成后 → 立刻检查新元素是否与旁边同级元素相邻
如果是 → 自动继续触发合成 → 分数倍率递增（×1, ×2, ×3...）
直到没有可连锁的相邻同级元素为止
```

**验证：** 触发连锁时显示 combo 计数，分数倍率正确递增。

---

### 3.6 实现吸附升级

```
每次合成/连锁后检查：
  是否有任何格子上的元素等级 = 核心等级？
  
如果是：
  触发吸附动画（元素飞向核心）
  核心等级 +1
  切换核心的视觉样式
  该格子变为空格
  
注意：任何圈层的同级元素都能被吸附，不限于内圈
```

**验证：** 合成出与核心同级的元素后，元素飞向核心，核心升级。

---

### 3.7 实现游戏结束

```
每次分裂前检查：
  所有 37 个格子是否全部被占满？
  
如果是：
  停止游戏循环
  显示游戏结束界面（核心等级 + 最终得分 + "再来一局"按钮）
```

**验证：** 填满棋盘后正确触发游戏结束，"再来一局"按钮能重置游戏。

## 第四阶段：视觉与粒子效果

### 4.1 元素视觉实现

有两种方式，**建议第一版先用方式 B**：

**方式 A：图片贴图（最终效果好）**
- 在 Figma 中画好 10 个等级的元素，导出为 PNG
- 放入 `images/` 文件夹
- 在 Canvas 中用 `drawImage` 绘制

**方式 B：代码绘制（推荐先用这个）**
- 用 Canvas API 直接画彩色圆形，不同等级用不同颜色区分
- 快速验证玩法没问题后，再替换为 Figma 导出的精美图片

> 💡 颜色参考见设计文档附录"元素颜色参考值"。

---

### 4.2 粒子效果系统

粒子效果的原理很简单：维护一个数组，每帧更新位置并画出来。

**粒子数据结构：**

```javascript
particle = {
  x, y,        // 位置
  vx, vy,      // 速度（每帧移动量）
  radius,      // 半径
  color,       // 颜色
  alpha,       // 透明度（0-1，每帧递减）
  life         // 剩余生命周期（帧数，归零后移除）
}
```

**四种粒子效果：**

| 效果 | 触发时机 | 说明 |
|------|----------|------|
| 合成爆发 | 合成发生时 | 在合成位置爆发一圈粒子，颜色与新元素一致 |
| 吸附尾迹 | 吸附动画时 | 元素飞向核心的路径上留下粒子尾迹 |
| Combo 爆发 | 连锁时 | 粒子数量和范围随 combo 数增大 |
| 元素装饰 | 常态 | 每个元素周围 1-3 个微小粒子缓慢漂浮 |

**Cursor 提示词示例：**
```
请在 particles.js 中实现粒子系统。每个粒子有 x, y, vx, vy,
radius, color, alpha, life 属性。提供 spawn 函数（在指定位置
生成多个粒子）、update 函数（每帧更新位置和透明度）、draw 函数
（渲染所有活着的粒子）。同屏粒子总数不超过 200 个。
```

> ⚠️ **性能注意：** 同屏粒子数不超过 200 个。元素装饰粒子每格 1-3 个，37 格最多约 100 个，加上合成爆发的临时粒子，总量可控。

## 第五阶段：打磨与测试

### 5.1 计分系统

在 `js/score.js` 中实现：

```javascript
// 基础分
mergeScore = newLevel * 10

// Combo 加成（第 n 次连锁 × n）
comboScore = mergeScore * comboCount

// 吸附升级奖励
absorbBonus = newCoreLevel * 100

// 最终得分累加
totalScore += mergeScore + comboScore + absorbBonus
```

---

### 5.2 音效（可选）

微信小游戏支持音频播放。可以添加简单的合成音效和吸附音效。

推荐免费音效资源：[freesound.org](https://freesound.org)

---

### 5.3 真机测试清单

在微信开发者工具中点击"预览"（不是模拟器），用微信扫码在真实手机上测试。

**必须验证的关键点：**

- [ ] 触摸响应是否准确（点击格子能否正确触发合成）
- [ ] 旋转是否流畅（不卡顿）
- [ ] 粒子效果是否正常
- [ ] 格子大小在小屏手机上也能点到
- [ ] 游戏结束和重新开始是否正常
- [ ] 长时间游戏（10分钟以上）是否有内存泄漏导致的卡顿

> ⚠️ **常见问题：** 如果触摸位置不准，通常是因为 Canvas 坐标和触摸坐标的缩放比例不对。让 AI 帮你检查 `canvas.width / canvas.clientWidth` 的比例是否正确处理。

## 第六阶段：发布上线

### 6.1 准备审核材料

微信小游戏上线需要以下材料：

| 材料 | 说明 | 准备时间 |
|------|------|----------|
| 软件著作权证书（软著） | 上线前置条件，审批周期较长 | **提前 1-3 个月申请** |
| 游戏截图 | 至少 3 张游戏内截图 | 开发完成后截图 |
| 游戏介绍文案 | 简短的游戏描述 | 提前准备 |
| 游戏类目 | 选择"休闲益智" | 注册时选择 |

> ⚠️ **软著要尽早申请！** 软著是上线的前置条件，审批周期较长。建议在开发第二阶段时就同步提交软著申请。可以在中国版权保护中心网站申请。

---

### 6.2 上传代码

1. 在微信开发者工具中点击"上传"按钮
2. 填写版本号（如 `1.0.0`）和备注
3. 上传成功后，登录 [mp.weixin.qq.com](https://mp.weixin.qq.com)
4. 在"版本管理"中找到刚上传的版本
5. 点击"提交审核"，填写审核信息
6. 等待审核通过（通常 1-3 个工作日）
7. 审核通过后，点击"发布"即可上线
